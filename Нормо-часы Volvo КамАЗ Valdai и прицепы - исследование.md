# Нормо-часы Volvo, КамАЗ, Valdai и прицепы

Дата проверки: `2026-03-16`

## Цель

Найти источники нормо-часов для:

- `Volvo`
- `КамАЗ`
- `Valdai`
- `прицепов`

и понять, что можно безопасно использовать в MVP для автоматического анализа заказ-нарядов.

Проверка делалась минимум по двум направлениям для каждой группы:

- официальный или близкий к OEM источник
- независимый коммерческий источник / публичная база / каталог норм

## Ключевой итог

На дату проверки `2026-03-16` в открытом интернете удалось надёжно подтвердить следующее:

- для `КамАЗ` есть современное официальное подтверждение существования норм времени по `VIN` и операциям, а также есть публичный исторический сборник с конкретными значениями для `КамАЗ-5320`
- для `Volvo` подтверждено наличие коммерческих truck labor time баз, включающих `Volvo`, но открытой проверяемой таблицы норм по операциям для текущих тягачей `Volvo Trucks` не найдено
- для `Valdai` найдены общие источники по `ГАЗ` и старые/общие сборники, но не найдена надёжная открытая таблица норм именно для актуального `Valdai / Valdai Next`
- для `прицепов` подтверждено наличие OEM-сервисных порталов и коммерческих labor time баз, включающих `trailers`, но открытой проверяемой таблицы норм по операциям для брендов вашего парка не найдено

Практический вывод:

- `КамАЗ` можно закладывать в систему как следующий реальный кандидат на подключение нормативов
- `Volvo`, `Valdai` и `прицепы` без платных или дилерских источников безопасно и полноценно закрыть нельзя

## 1. КамАЗ

## Что подтверждено официально

Официальная презентация экосистемы ПАО `КАМАЗ` подтверждает, что в портале есть:

- поиск норм времени по `VIN`
- код и наименование операции
- трудоёмкость операции
- калькулятор технического обслуживания и ремонта

Источник:

- `https://itorum.ru/wp-content/uploads/2024/11/04_%D0%AD%D0%BA%D0%BE%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0%20%D0%BF%D0%BE%D0%B4%D0%B4%D0%B5%D1%80%D0%B6%D0%BA%D0%B8%20%D0%B6%D0%B8%D0%B7%D0%BD%D0%B5%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE%20%D1%86%D0%B8%D0%BA%D0%BB%D0%B0%20%D0%B8%D0%B7%D0%B4%D0%B5%D0%BB%D0%B8%D0%B9%20%D0%9F%D0%90%D0%9E%20%E2%80%9C%D0%9A%D0%90%D0%9C%D0%90%D0%97%E2%80%9D.pdf`

Что это значит для проекта:

- у `КАМАЗ` есть современный нормативный контур
- правильный путь интеграции для production это не ручной поиск по интернету, а выгрузка или лицензируемый доступ к официальному источнику по `VIN`

## Что подтверждено вторым источником

Публичная страница `АвтоДилер` отдельно заявляет нормы времени для `КамАЗ` и регулярно обновляемую базу.

Источник:

- `https://autodealer.ru/times/kamaz`

Это не OEM-источник, но это второе независимое подтверждение, что нормо-часы по `КамАЗ` существуют и используются как отдельный коммерческий каталог.

## Что найдено в открытом виде с конкретными значениями

Найден публичный исторический сборник:

- `https://meganorm.ru/Data2/1/4293737/4293737849.pdf`

Это не современный дилерский источник и не подходит как единственный production-справочник для текущих моделей, но это реальный документ с конкретными нормами для `КамАЗ-5320`.

Примеры значений из раздела по дизельным грузовикам `МАЗ-500 / МАЗ-503 / КамАЗ-5320 / КрАЗ-256Б`:

- `КамАЗ-5320` снять двигатель без коробки передач:
  - электротехнические работы `0.38 чел.-ч`
  - слесарные работы `5.10 чел.-ч`
- установить двигатель без коробки передач:
  - электротехнические работы `0.52 чел.-ч`
  - слесарные работы `6.10 чел.-ч`
- снять и установить головку цилиндров: `0.80 чел.-ч`
- притереть клапаны: `2.60 чел.-ч`
- отрегулировать зазор клапанов: `0.55 чел.-ч`
- снять и установить коромысла вместе с валами и штангами: `0.93 чел.-ч`
- заменить поршневые кольца: `6.20 чел.-ч`
- заменить вкладыши шатунных подшипников: `1.85 чел.-ч`
- заменить вкладыши коренных подшипников: `2.00 чел.-ч`

## Вывод по КамАЗ

Для `КамАЗ` нормативное направление подтверждено лучше всего.

Рекомендация:

- собирать современный справочник через официальный контур `КАМАЗ` или лицензируемую выгрузку
- исторический публичный сборник использовать только как временный reference layer и только с явной пометкой `legacy / historical`

## 2. Volvo

## Что найдено по официальным источникам

На публичном сайте `Volvo Trucks` найден открытый официальный контур по обслуживанию и maintenance intervals, но не найдена открытая таблица норм времени по операциям.

Источник:

- `https://www.volvotrucks.us/parts-and-services/services/service-and-maintenance-intervals/`

Это подтверждает наличие официальной сервисной документации и регламентов обслуживания, но не даёт публичной выгрузки labor times.

## Что найдено вторым источником

Найден коммерческий источник truck labor times, где явно присутствует `Volvo` и отдельный блок по `Trailers`.

Источник:

- `https://repair.diesellaptops.com/truck-labor-times/`

По содержимому страницы видно, что база покрывает:

- `Engine – Volvo`
- `Trailers (1980-2020)`

Это важное второе подтверждение:

- labor times по `Volvo` для грузовой техники существуют в коммерческих системах
- но открытой таблицы конкретных норм для ваших текущих тягачей `Volvo` публично не найдено

## Что НЕ удалось надёжно подтвердить

На дату проверки `2026-03-16` не найдена открытая, верифицируемая и пригодная для production таблица вида:

- `VIN/модель Volvo -> операция -> нормо-часы`

Поэтому безопасно импортировать публичный `Volvo`-справочник в MVP сейчас нельзя.

## Вывод по Volvo

Для `Volvo` нужно получать нормы из одного из источников:

- официальный дилерский / OEM-контур Volvo Trucks
- коммерческая heavy-duty база labor times

Без этого любые интернет-таблицы будут неполными или сомнительными.

## 3. Valdai

## Что найдено в открытых источниках

По `Valdai / Valdai Next` публично находятся:

- официальные продуктовые страницы
- руководства по эксплуатации
- общие ремонтные руководства и книжные каталоги

Официальная продуктовая страница:

- `https://gaz.ru/trucks/valdai/`

Коммерческий каталог норм по марке `ГАЗ`:

- `https://motordata.net/models-normy-vremeni/gaz/`

## Что удалось перепроверить вторым проходом

Во втором проходе поиска нашлись:

- общие руководства и книги по ремонту `Valdai Next`
- старые типовые нормы по грузовикам `ГАЗ`

Но не нашлась надёжная открытая таблица норм именно для актуального `Valdai / Valdai Next`, которую можно безопасно импортировать как production-catalog.

Важно:

- старые сборники по грузовикам `ГАЗ` не равны нормативам для современного `Valdai`
- подмена `общих норм ГАЗ` на `Valdai Next` приведёт к высоким рискам ложного анализа

## Вывод по Valdai

На дату проверки `2026-03-16` проверяемый public-ready справочник для `Valdai` не найден.

Для корректной автоматизации нужен один из вариантов:

- дилерский / OEM источник по `Valdai`
- коммерческая база labor times, где модель `Valdai` или `Valdai Next` явно присутствует
- ваш внутренний сервисный регламент по этим машинам

## 4. Прицепы

## Что подтверждено официально

Для `Schmitz Cargobull` найден официальный сервисный контур:

- `https://www.cargobull-serviceportal.de/applications/serviceportal/default.aspx`
- `https://www.cargobull.com/de/services/trailer-services/my-cargobull`

Это подтверждает, что OEM-сервисная инфраструктура по прицепам есть.

## Что подтверждено вторым источником

Коммерческая heavy-duty labor time база `Diesel Repair` явно показывает блок:

- `Trailers (1980-2020)`

Источник:

- `https://repair.diesellaptops.com/truck-labor-times/`

Это второе независимое подтверждение, что labor times по прицепам существуют как отдельный класс данных.

## Что не найдено в открытом интернете

Для брендов вашего прицепного парка:

- `SCHMITZ`
- `SERIN`
- `ORTHAUS`
- `BONUM`
- `KOLUMAN`
- `Krone`
- `ТОНАР`

не найдена открытая и проверяемая таблица:

- `бренд/модель прицепа -> операция -> нормо-часы`

Отдельно важно:

- у прицепов значительная часть работ нормативно зависит не только от бренда прицепа, но и от компонентов `BPW / SAF / WABCO / Knorr-Bremse / оси / тормозная система / подвеска / надстройка`
- без component-level справочников анализ будет грубым и часто ошибочным

## Вывод по прицепам

Для прицепов безопасный production-подход такой:

- сначала собирать нормы по `Schmitz`, потому что по нему подтверждён OEM service portal
- параллельно строить нормирование не только по бренду прицепа, но и по компонентам
- для остальных брендов ждать OEM-источники, внутренние регламенты или платную trailer labor time базу

## 5. Что реально можно делать в системе уже сейчас

## Наиболее реалистичный порядок подключения

### Шаг 1. КамАЗ

Подключать первым после `Dong Feng`, потому что:

- есть официальное подтверждение норм времени по `VIN`
- есть вторичный коммерческий источник
- есть хотя бы исторический публичный reference со значениями

### Шаг 2. Volvo

Подключать вторым, но только после получения:

- дилерского / OEM доступа
- либо коммерческой heavy-duty labor time базы

### Шаг 3. Schmitz / прицепы

Подключать после решения по component-level нормам:

- оси
- тормоза
- пневматика
- подвеска
- кузовные и надстроечные работы

### Шаг 4. Valdai

Подключать после нахождения model-specific норм.

Сейчас делать автоматический импорт `Valdai`-норм из открытого интернета не рекомендуется.

## 6. Решение для MVP

С учётом качества найденных источников, для MVP рационально зафиксировать такую политику:

- `Dong Feng` использовать автоматически уже сейчас
- `КамАЗ` готовить как следующий справочник для подключения
- `Volvo` пометить как `нормативы доступны только после получения внешней базы`
- `Valdai` пометить как `публичный подтверждённый справочник не найден`
- `прицепы` пометить как `нужен отдельный trailer/component labor guide`

## 7. Что нужно запросить у бизнеса или у сервисов

Чтобы закрыть тему качественно, нужно получить хотя бы один из наборов данных:

- официальный экспорт норм по `КАМАЗ`
- дилерский или коммерческий экспорт labor times по `Volvo`
- внутренние регламенты по `Valdai`
- OEM или component-based справочник по прицепам

## Финальный вывод

На дату `2026-03-16` корректно утверждать следующее:

- `КамАЗ` реально можно двигать в работу следующим после `Dong Feng`
- для `Volvo` нормы существуют, но в открытом интернете не найдена безопасная production-таблица
- для `Valdai` не найден подтверждённый model-specific справочник
- для `прицепов` есть сервисные порталы и labor time базы, но не найдена открытая детальная таблица по брендам вашего парка

Если внедрять это без внешнего источника, качественный анализ заказ-нарядов будет надёжным только для:

- `Dong Feng`
- частично и с оговорками для `КамАЗ` на historical/reference уровне

Для остального парка сейчас правильнее показывать в системе:

- `норматив недоступен`
- `нужен внешний справочник`
- `сопоставление невозможно без model-specific labor guide`
