# Сводка по источнику заказ-нарядов `2025 для ИИ.xlsx`

## Источник

- Оригинальный файл: `/Users/svetlanasamojlova/Library/Mobile Documents/com~apple~CloudDocs/2025 для ИИ.xlsx`
- Рабочая копия в проекте: `Заказ-наряды/2025 для ИИ.xlsx`

## Что это за файл

Это Excel-выгрузка с одним листом `Лист_1`, содержащая строки по заказ-нарядам, расходным материалам, поставщикам, транспортным средствам и номенклатуре.

По структуре это не набор PDF, а табличный реестр хозяйственных операций, который можно использовать как исходный датасет для:

- анализа истории ремонтов и закупок
- извлечения справочников поставщиков, колонн, моделей ТС и номенклатуры
- сопоставления заказ-нарядов с будущими PDF-документами
- подготовки импорта исторических данных в систему

## Что видно по структуре первых строк

Примеры колонок:

- `ТС.Государственный номер`
- `ТС.Вид ТС`
- `Период`
- `Пробег`
- `Поставщик`
- `Колонна`
- `Регистратор`
- `Номенклатура.Автогрупп (Номенклатура)`
- `Номенклатура.Артикул`
- `Статья затрат`
- `ТС.Модель`
- `Номенклатура`

По первым строкам встречаются записи вида:

- `Заказ-наряд 000005020 от 17.04.2025 23:59:59`
- `Заказ-наряд 000006744 от 30.05.2025 23:59:59`
- `Выдача расходных материалов 000000309 от 12.02.2025 23:59:59`

## Что это даёт для проекта

Файл полезен как промежуточный источник предметных данных даже без PDF:

- из поля `Регистратор` можно извлекать номер и дату заказ-наряда
- из полей ТС можно формировать предварительный реестр машин
- из полей поставщика и статьи затрат можно строить справочники сервисов и категорий расходов
- по номенклатуре можно анализировать материалы и запчасти
- по пробегу можно связывать ремонт с состоянием автомобиля на момент обращения

## Ограничения

- это не оригиналы заказ-нарядов, а выгрузка из учётной системы
- в одном наборе могут смешиваться ремонты, запчасти и выдача материалов
- потребуется очистка данных и правила классификации строк
- даты Excel хранятся в числовом формате и при импорте должны конвертироваться

## Рекомендация по использованию

Для MVP этот файл стоит рассматривать как дополнительный источник исторических данных и справочников, а не как замену PDF. Основной сценарий остаётся таким:

- пользователь загружает PDF заказ-наряда
- система извлекает поля из PDF
- при наличии данных в выгрузке можно делать сверку по номеру заказ-наряда, машине, поставщику и номенклатуре
