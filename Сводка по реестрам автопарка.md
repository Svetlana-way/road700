# Сводка по реестрам автопарка

## Источники

- Оригинал грузовиков: `/Users/svetlanasamojlova/Library/Mobile Documents/com~apple~CloudDocs/Грузовики Парк.xls`
- Оригинал прицепов: `/Users/svetlanasamojlova/Library/Mobile Documents/com~apple~CloudDocs/Прицепы Парк.xls`
- Рабочие копии в проекте:
  - `Реестр автомобилей/Грузовики Парк.xls`
  - `Реестр автомобилей/Прицепы Парк.xls`

## Назначение

Эти два файла являются исходными реестрами автопарка для первичной загрузки машин в систему.

Они нужны для:

- заполнения таблицы транспортных средств
- проверки существования машины при загрузке ремонта
- распределения машин по колоннам и механикам
- подготовки разграничения доступа сотрудников

## Что удалось подтвердить по структуре

Файлы находятся в старом формате Excel `.xls`.

Несмотря на бинарный формат, из них читаются заголовки и значения, достаточные для проектирования импорта.

### Грузовики

Подтверждённые поля:

- `ID в CARGO.RUN`
- `Марка`
- `Год выпуска`
- `Привязанные прицепы`
- `Текущий водитель 2`
- `Колонна`
- `Механик`
- `Дата последних координат`

По содержимому также явно встречаются:

- государственные номера
- VIN
- комментарии по машине

Примеры значений:

- госномер: `н443рк/716`, `к646рк/716`, `о750сх/716`
- VIN: встречаются строки вида `X9PRG20A4MW139382`
- колонны: `Колонна 4`, `Колонна 5`, `Колонна 13`
- механики и ответственные: ФИО сотрудников в явном виде

### Прицепы

Подтверждённые поля:

- `ID в CARGO.RUN`
- `Марка`
- `Год выпуска`
- `Привязанные грузовики`
- `Механик`
- `Комментарий`

По содержимому также явно встречаются:

- государственные номера
- VIN
- тип или категория прицепа

Примеры значений:

- марки и типы: `SCHMITZ`, `Рефрижератор`, `Тент`
- VIN: встречаются строки вида `X1Y908400R3271402`
- комментарии: `# переоформлен 01.12.22`, `1394ва/16 ru от 13.11.2020 #`

## Вывод для модели данных

На основании этих реестров в MVP можно уверенно закладывать импорт следующих полей для таблицы ТС:

- внешний идентификатор из CARGO.RUN
- тип ТС: грузовик или прицеп
- госномер
- VIN
- марка
- год выпуска
- колонна
- механик
- комментарий
- привязанные единицы техники

Дополнительно для грузовиков:

- текущий водитель
- дата последних координат

## Практическая рекомендация

Импорт стоит делать через два отдельных шаблона:

- импорт грузовиков
- импорт прицепов

После импорта обе сущности лучше хранить в одной таблице `vehicles` с полем `vehicle_type`, либо в одной логической модели с типом `truck` и `trailer`.

## Ограничения

- формат `.xls` старый, поэтому для надёжного импорта потребуется библиотека чтения Excel старого формата
- итоговый список колонок лучше подтвердить повторно после открытия файлов в полноценном Excel-ридере
- возможны скрытые колонки или особенности форматирования, которые бинарный просмотр не показывает полностью
