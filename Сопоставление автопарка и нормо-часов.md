# Сопоставление автопарка и нормо-часов

## Источники

- Реестр грузовиков: `Реестр автомобилей/Грузовики Парк.xls`
- Реестр прицепов: `Реестр автомобилей/Прицепы Парк.xls`
- Справочник нормо-часов: `Норма часов Донг Фенг 2025 год.xlsx`

## Ключевой вывод

Текущий справочник нормо-часов покрывает только часть автопарка и не содержит явной зависимости нормо-часов от возраста автомобиля, года выпуска, модификации по VIN, комплектации, типа кузова или других эксплуатационных параметров.

В справочнике Dong Feng норматив задаётся по коду работы и описанию работы.

То есть в текущем источнике норма выглядит как:

- `код работы`
- `описание работы`
- `категория узла`
- `норматив времени в часах`

Но не как:

- `марка + модель + год + модификация + работа -> отдельная норма`

## Зависимость нормо-часов от возраста, модификации и других параметров

### Что источник поддерживает явно

Источник поддерживает только такие оси различия:

- код операции
- описание операции
- категория узла
- иногда альтернативное русское описание

### Что источник не поддерживает явно

В текущем файле не найдено отдельных колонок или отдельных таблиц для:

- года выпуска
- возраста автомобиля
- модификации по VIN
- двигателя
- коробки передач
- колесной формулы
- типа кузова
- комплектации
- экологического класса
- рынка/страны исполнения

Следовательно:

- зависимость нормо-часов от возраста машины в текущем справочнике не предусмотрена явно
- зависимость от модификации машины тоже не предусмотрена явно
- если такие различия важны, их придётся вводить дополнительными правилами или отдельным справочником

## Какие параметры всё равно нужно учитывать в анализе заказ-нарядов

Даже если справочник не даёт отдельной нормы по возрасту и модификации, для качественного анализа заказ-нарядов в системе нужно учитывать следующие параметры техники:

### 1. Марка и семейство модели

Критично, потому что сейчас нормы есть только для Dong Feng.

Минимальные сегменты:

- `DFH4180 / DONGFENG`
- `VOLVO`
- `КАМАЗ`
- `Valdai`
- прицепы отдельно

### 2. Тип техники

Минимальные типы:

- тягач седельный
- фургон реф
- фургон тент
- прицеп тент
- прицеп рефрижератор

### 3. VIN-префикс / производственная серия

Для Dong Feng в автопарке сейчас хорошо выделяется серия:

- `LGAG3DV2`

Это полезно как технический признак семейства машины и возможной будущей детализации нормативов.

### 4. Год выпуска

Даже если нормы не разделены по годам, год нужен для аналитики как фактор:

- износа
- повторяемости ремонтов
- вероятности тяжёлых ремонтов
- отклонения факта от нормы

### 5. Вид работы и код работы

Это основной слой для нормо-часов.

Для каждой строки заказ-наряда надо хранить:

- исходное название работы из документа
- распознанный код работы
- связанный код нормы
- справочное количество часов
- фактическое количество часов
- способ сопоставления

### 6. Тип документа и качество OCR

Для анализа качества отклонений важно понимать:

- PDF это или скан/фото
- насколько уверенно распознан документ
- подтверждён ли матчинг работы вручную

## Практический вывод для MVP

Сейчас можно уверенно запускать такой режим:

- для `DFH4180` и `DONGFENG` пытаться автоматически сопоставлять работы со справочником Dong Feng
- считать отклонение фактических часов от нормы
- помечать подозрительные работы при превышении нормы более чем на `10%`

Но нельзя считать, что уже есть полное покрытие:

- всех брендов грузовиков
- всех прицепов
- возрастных различий
- модификаций и комплектаций

## Что нужно дополнительно для полноценного анализа

Чтобы действительно получить нормо-часы:

- для каждого вида автомобиля
- для каждого вида работ
- с учётом возраста и модификации

нужно дополнительно собрать:

- нормативы по `VOLVO`
- нормативы по `КАМАЗ`
- нормативы по `Valdai`
- нормативы по прицепам
- если существуют, отдельные нормативные таблицы по модификациям Dong Feng
- если существуют, правила применимости норм по VIN-серии, двигателю, КПП и исполнению шасси

## Итог

На текущем наборе данных корректно утверждать следующее:

- предусмотренные нормо-часы уже найдены и могут использоваться для Dong Feng-тягачей
- в текущем источнике они задаются по виду работы, а не по возрасту машины
- явной зависимости от модификации и возраста в файле не обнаружено
- для качественного анализа заказ-нарядов нужно учитывать как минимум:
  - марку/семейство
  - тип техники
  - VIN-серию
  - год выпуска
  - код и вид работы
  - качество распознавания документа

Поэтому для текущего MVP корректная стратегия такая:

- использовать Dong Feng-нормы там, где они действительно применимы
- не распространять их автоматически на Volvo, КамАЗ, Valdai и прицепы
- хранить признаки техники и матчинга так, чтобы позже можно было добавить дополнительные нормативные справочники
