# План 1

## Task List

1. `Background Job` модель и миграция
Файлы: `backend/app/models`, `backend/alembic`
Критерий готовности: в БД есть таблица фоновых задач со статусами `queued / processing / completed / failed / retry`, задача создается и читается из ORM.

2. Сервис управления задачами
Файлы: `backend/app/services`, `backend/app/schemas`
Критерий готовности: есть единый сервис `create/start/complete/fail/retry job`, статус меняется только через него.

3. Перевод загрузки документа на job flow
Файлы: `backend/app/api/documents.py`, `backend/app/services/document_processing.py`
Критерий готовности: после загрузки документа создается job, а не `BackgroundTasks`; документ получает связанный `job_id`.

4. Worker-процесс для обработки документов
Файлы: `backend/app/scripts`, `docker-compose.server.yml`, `Dockerfile.app`
Критерий готовности: отдельный воркер поднимается в compose и забирает `queued` задачи без участия API-процесса.

5. Логирование пайплайна обработки
Файлы: `backend/app/services/document_processing.py`
Критерий готовности: в логах видны этапы `start -> extract -> parse -> match -> save -> report`, при ошибке есть traceback и `job.error_message`.

6. API статуса и перезапуска задачи
Файлы: `backend/app/api`, `backend/app/api/router.py`
Критерий готовности: работают `GET /api/jobs/{id}` и `POST /api/jobs/{id}/retry`.

7. Отображение статуса обработки во фронте
Файлы: `frontend/src/App.tsx`
Критерий готовности: пользователь видит `В очереди / Обрабатывается / Ошибка / Готово`, при ошибке доступна кнопка повтора.

8. Backend integration tests на ключевой сценарий
Файлы: `backend/tests`
Критерий готовности: есть тесты на логин, загрузку документа, создание job, успешную обработку, открытие endpoint ремонта и наличие `executive_report`.

9. Усиление predeploy-проверки
Файлы: `scripts/predeploy-usecase-check.sh`, `.github/workflows/ci.yml`
Критерий готовности: перед деплоем автоматически проверяется не только сборка, но и основной сценарий открытия отчета.

10. Введение роутинга на фронте
Файлы: `frontend/package.json`, `frontend/src/main.tsx`, `frontend/src/App.tsx`
Критерий готовности: есть отдельные маршруты для списка техники, карточки машины, карточки ремонта и отчета.

11. Вынос API-клиента из UI
Файлы: `frontend/src`
Критерий готовности: `fetch`-логика вынесена в отдельный слой `shared/api`, UI-компоненты не собирают запросы вручную.

12. Разрезание `App.tsx` на страницы
Файлы: `frontend/src/App.tsx`
Критерий готовности: карточка ремонта, executive report и карточка транспорта вынесены в отдельные компоненты/страницы, `App.tsx` перестает быть центром всей системы.

## Первая пачка работ

1. `background_job` + миграция
2. worker + compose service
3. перевод `documents.py` на создание job
4. логирование ошибок обработки
5. `GET /api/jobs/{id}` и `POST /api/jobs/{id}/retry`
6. backend integration test на открытие отчета по ремонту
